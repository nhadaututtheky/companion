# Phase 3: Debate Mode Upgrade

## Goal
Port Coordinator Mode patterns (system prompt, task-notification format, scratchpad, independent verification) into the existing Debate Engine. Pattern-level adoption, NOT code dependency.

## Current State
- Debate engine uses simple parallel agent execution with no coordinator
- System prompts are role-based but lack meta-orchestration
- Verdict is generated by a "judge" with full debate context (no independence)
- No scratchpad or accumulated shared context between rounds

## Changes
1. **Coordinator prompt**: Add coordinator system that synthesizes between rounds
2. **Task-notification XML format**: Structure agent responses for cleaner parsing + context
3. **Scratchpad**: Accumulated key points shared between rounds (reduces context bloat)
4. **Independent verdict**: Verifier gets summary only, not full transcript (avoids bias)

- [ ] Task 2 — Implement task-notification XML format for agent messages
- [ ] Task 3 — Add scratchpad (accumulated key points per round)

## Acceptance Criteria
- [ ] Coordinator synthesizes after each round (progress summary + direction)
- [ ] Agent responses wrapped in `<task-notification>` XML for structured context
- [ ] Scratchpad grows each round with key arguments (caps at ~2000 tokens)
- [ ] Judge gets scratchpad + last round only (not full 5-round transcript)
- [ ] No regression on existing debate formats (pro_con, red_team, review, brainstorm)
- [ ] Zero type errors
